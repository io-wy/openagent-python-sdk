# Security Policy

## Supported Versions

Only the latest stable release receives security fixes.

| Version | Supported |
|---------|-----------|
| 0.3.x   | Yes       |
| < 0.3   | No        |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Please report vulnerabilities by emailing **astro_air@126.com** with:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Affected versions

You will receive an acknowledgement within **48 hours** and a status update within **7 days**. If a fix is warranted, we aim to release it within **30 days** of confirmation.

Once a fix is released, the issue may be disclosed publicly with appropriate credit unless you request otherwise.
